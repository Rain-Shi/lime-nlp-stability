9  Perturbation Stability: Comparing LIME and SHAP for Logistic Regression and BERT

9.1 Goal

This notebook compares the stability of model explanations under minor input perturbations using two explainability techniques: - LIME - SHAP

Across two models: - TF-IDF + Logistic Regression (interpretable linear baseline) - Fine-tuned BERT (complex deep contextual model)

9.2 1. Setup and Load

import pandas as pd
import numpy as np
import random
import matplotlib.pyplot as plt
import seaborn as sns

random.seed(42)

col_names = ["id", "entity", "sentiment", "tweet"]
val = pd.read_csv("data/twitter_validation.csv", header=None, names=col_names)
val = val.dropna(subset=["tweet"])
val = val[val["tweet"].str.strip().astype(bool)]
val = val.sample(1, random_state=42)
tweet_text = val.iloc[0]["tweet"]

9.3 2. Perturbation & Similarity Function

def random_deletion(text, p=0.2):
    words = text.split()
    new_words = [w for w in words if random.random() > p]
    return " ".join(new_words) if new_words else random.choice(words)

def jaccard(set1, set2):
    return len(set1 & set2) / len(set1 | set2)

9.4 3. LIME Stability Comparison

9.4.1 3.1 Logistic Regression

from sklearn.pipeline import make_pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from lime.lime_text import LimeTextExplainer

# Train model
train = pd.read_csv("data/twitter_training.csv", header=None, names=col_names)
lr_model = make_pipeline(TfidfVectorizer(), LogisticRegression())
lr_model.fit(train["tweet"], train["sentiment"])

# Explainer
explainer = LimeTextExplainer(class_names=["Positive", "Neutral", "Negative", "Irrelevant"])

def get_lime_words(text, model_fn):
    exp = explainer.explain_instance(text, model_fn, num_features=10, top_labels=1)
    label = exp.top_labels[0]
    return set([w for w, _ in exp.as_list(label=label)])

orig_words_lr = get_lime_words(tweet_text, lr_model.predict_proba)
scores_lime_lr = [jaccard(orig_words_lr, get_lime_words(random_deletion(tweet_text), lr_model.predict_proba)) for _ in range(5)]

9.4.2 3.2 BERT Model

import torch
from transformers import AutoTokenizer, AutoModelForSequenceClassification

bert_path = "./scripts/bert_model4"
bert_model = AutoModelForSequenceClassification.from_pretrained(bert_path)
bert_tokenizer = AutoTokenizer.from_pretrained(bert_path)
bert_model.eval()


def predict_fn(texts):
    inputs = bert_tokenizer(texts, return_tensors="pt", padding=True, truncation=True, max_length=128)
    inputs = {k: v.to(bert_model.device) for k, v in inputs.items()}
    with torch.no_grad():
        logits = bert_model(**inputs).logits
    return torch.nn.functional.softmax(logits, dim=-1).detach().cpu().numpy()

orig_words_bert = get_lime_words(tweet_text, predict_fn)
scores_lime_bert = [jaccard(orig_words_bert, get_lime_words(random_deletion(tweet_text), predict_fn)) for _ in range(5)]

9.5 4. SHAP Stability Comparison

9.5.1 4.1 Logistic Regression

import shap

explainer_lr = shap.Explainer(lr_model.predict_proba, lr_model.named_steps["tfidfvectorizer"].transform)
shap_vals_orig = explainer_lr([tweet_text])
words_orig = set(shap_vals_orig.data[0].split())
scores_shap_lr = [
    jaccard(words_orig, set(explainer_lr([random_deletion(tweet_text)]).data[0].split()))
    for _ in range(5)
]

9.5.2 4.2 BERT

explainer_bert = shap.Explainer(predict_fn, bert_tokenizer)
shap_vals_orig_bert = explainer_bert([tweet_text])
words_orig_bert = set(shap_vals_orig_bert.data[0].split())
scores_shap_bert = [
    jaccard(words_orig_bert, set(explainer_bert([random_deletion(tweet_text)]).data[0].split()))
    for _ in range(5)
]

9.6 5. Combined Visualization

df = pd.DataFrame({
    "Model": ["LR (LIME)"] * 5 + ["BERT (LIME)"] * 5 + ["LR (SHAP)"] * 5 + ["BERT (SHAP)"] * 5,
    "Similarity": scores_lime_lr + scores_lime_bert + scores_shap_lr + scores_shap_bert
})

plt.figure(figsize=(10, 6))
sns.boxplot(data=df, x="Model", y="Similarity", palette="pastel")
sns.stripplot(data=df, x="Model", y="Similarity", color="black", alpha=0.6)
plt.title("Perturbation Stability Comparison (LIME vs SHAP)")
plt.ylabel("Jaccard Similarity")
plt.xticks(rotation=20)
plt.tight_layout()
plt.show()

9.7 Summary

  • Logistic Regression consistently shows higher Jaccard similarity across LIME and SHAP, suggesting more stable explanations.
  • BERT exhibits more variance and lower average similarity, especially under SHAP.
  • These results reinforce the trade-off: more powerful models tend to have less robust interpretability under input changes.
  • This experiment confirms the trend observed in both explanation methods (LIME & SHAP).

Overall, explanation stability should be considered when deploying complex NLP models in sensitive applications.
